# 🔒 Responsible Disclosure Policy

## Sanitization Standards

All content in this repository follows strict sanitization guidelines:

- ❌ No live IP addresses from production systems
- ❌ No production credentials, API keys, or tokens
- ❌ No private keys or certificates from real systems
- ❌ No active exploit payloads targeting live systems
- ❌ No personally identifiable information (PII) of third parties

## What IS Published

- ✅ Educational CTF writeups using publicly available challenges
- ✅ Conceptual exploit demonstrations on intentionally vulnerable apps (DVWA, WebGoat, etc.)
- ✅ Sanitized scripts with placeholder IP addresses (e.g., `10.10.10.x`, `TARGET_IP`)
- ✅ Tool documentation and methodology notes

## Lab Environments Used

All hands-on labs were conducted in:
- **TryHackMe** — Isolated VPN environments
- **HackTheBox** — Isolated VPN environments  
- **Local VMs** — Isolated VirtualBox/VMware environments
- **DVWA** — Locally hosted intentionally vulnerable web application
- **WebGoat** — Locally hosted OWASP learning platform

## Reporting Real Vulnerabilities

If you discover a real vulnerability, please follow responsible disclosure:
1. Contact the vendor/organization privately
2. Allow a reasonable remediation period (90 days recommended)
3. Coordinate public disclosure
