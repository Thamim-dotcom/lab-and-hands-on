## 🧠 Key Study Strategies

### 1. Active Recall > Passive Reading
Instead of re-reading notes, I used Anki flashcards for:
- Tool names and their purposes (Nmap flags, Metasploit commands)
- Attack technique definitions
- Protocol specifics (port numbers, headers, handshakes)

### 2. Hands-On First
For every module, I practiced in a lab environment BEFORE reading about it:
```
Read concept (15 min) → Practice in TryHackMe/iLabs (45 min) → Review → Repeat
```

### 3. Practice Exam Strategy
- Completed 100 practice questions daily in the final month
- Reviewed ALL incorrect answers with explanations
- Identified weak modules and double-down reviewed them
- Tracked accuracy by module using a spreadsheet

## 📋 Exam Day Strategy

**Exam Format:** 125 questions, 4 hours, Multiple Choice + Scenario-Based  

**My Approach:**
1. **First Pass (2 hrs):** Answer all confident questions, flag uncertain ones
2. **Second Pass (1 hr):** Review flagged questions, eliminate wrong answers
3. **Final Pass (1 hr):** Review flagged again, gut-check answers
4. **Never leave blank** — No penalty for wrong answers

**Time Management:**
```
125 questions × 4 hours = ~1.9 min per question
First pass: 2 hours (≈60% of questions answered confidently)
Second pass: 1 hour (flagged questions reviewed)
Final review: 1 hour (final check)
```

## 💡 Tips for Aspiring CEH Candidates

1. **Don't rely only on the courseware** — supplement with TryHackMe and HTB
2. **Do the iLabs** — EC-Council's labs are directly relevant to exam scenarios
3. **Focus on tools** — Know which tool does what, not just theory
4. **Read carefully** — Scenario questions are tricky, read every word
5. **Know your OSI model cold** — It underlies almost every question
6. **4 hours is enough** — Don't rush, you have time to review
